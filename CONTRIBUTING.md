# 路线数据贡献说明

欢迎补充或纠正 `wangshifu-data.json` 中的路线点。

贡献者只需提供：

- 视频链接或 BV 号；
- 任意一条你确认的信息，例如地点、当天里程、吃了什么，或者现有内容哪里不对。

不知道经纬度、准确时间点、可信度或摘要都没关系，维护者会负责查证、定位和整理成统一格式。视频时间点、路牌截图、口播等依据属于加分项，不是提交门槛。

可信度建议：

- `标题确认`：视频标题直接出现地点或里程。
- `画面确认`：路牌、码表或地图画面可以确认。
- `口播确认`：视频口播明确说明。
- `路线推断`：仅能根据前后城市推断，必须明确标注。

不要把两点之间的直线距离当作真实骑行里程，也不要把公开视频落脚点描述成 GPS 实时位置。

最省事的方式是使用网页右上角的“补路线点”，写一句你发现的内容后复制提交。熟悉 GitHub 的贡献者也可以直接修改 JSON 或提交 Pull Request。

## 点赞采纳

- 每条建议作为一个 GitHub Issue，观众用 👍 reaction 投票。
- 同一期视频出现多条建议时，优先核验点赞较高的内容。
- 达到 3 个 👍 的建议会自动标记为 `票选-优先`。
- 点赞代表核验优先级，不替代事实核查；维护者确认视频内容后再写入正式数据。
